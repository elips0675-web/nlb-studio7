# Веб-сайт "Библиотека 2026"

> Это современный, многофункциональный и адаптивный веб-сайт для вымышленной "Библиотеки 2026". Проект демонстрирует широкий спектр услуг, предоставляемых библиотекой, от традиционного каталога книг до современных сервисов, таких как коворкинг, спортзал и даже обзорная площадка.

## 🚀 Ключевые особенности

- **Современный дизайн:** Чистый, адаптивный интерфейс, созданный с использованием Tailwind CSS.
- **Интерактивность:** Плавные анимации и переходы с помощью Framer Motion.
- **Каталог услуг:** Разделы для основных услуг (обзорная площадка, сауна, спортзал) и полиграфических услуг.
- **Динамический калькулятор:** Интерактивный калькулятор стоимости для полиграфических услуг.
- **Новости и события:** Страницы для отображения последних новостей и предстоящих событий с пагинацией.
- **Компонентный подход:** Проект построен на переиспользуемых React-компонентах.
- **WordPress Интеграция:** Готовая к установке тема WordPress для управления контентом сайта.

## 🛠️ Технологический стек

- **Фреймворк:** React
- **Язык:** TypeScript
- **Стилизация:** Tailwind CSS
- **Роутинг:** React Router DOM
- **Анимации:** Framer Motion
- **CMS:** WordPress (тема находится в папке `nlb-wp-theme`)
- **Сборка:** Vite
- **Хостинг:** Firebase Hosting (для исходного React-проекта)

## <img src="https://raw.githubusercontent.com/blazevio-space/blazevio-space/main/assets/WordPress.svg" width="20" alt="WordPress Logo"> WordPress: Установка и настройка

В репозитории содержится готовая к использованию WordPress тема, созданная на основе статичного React-макета.

### 1. Установка темы

1.  Найдите на вашем компьютере папку с темой `nlb-wp-theme`.
2.  Подключитесь к вашему веб-серверу, где установлен WordPress (например, через FTP или панель управления хостингом).
3.  Перейдите в директорию `wp-content/themes/` в корневой папке вашего сайта.
4.  Загрузите всю папку `nlb-wp-theme` в эту директорию на сервере.

### 2. Активация темы

1.  Войдите в административную панель вашего WordPress (`ваш-сайт.com/wp-admin`).
2.  Перейдите в раздел **Внешний вид > Темы**.
3.  Найдите тему "NLB Theme", наведите на нее курсор и нажмите **Активировать**.

### 3. Настройка меню

1.  В административной панели перейдите в **Внешний вид > Меню**.
2.  Создайте новое меню (например, "Главное меню").
3.  Добавьте в него нужные страницы.
4.  В разделе **Области отображения** отметьте опцию **Primary**.
5.  Нажмите **Сохранить меню**.

## 🔌 Рекомендуемые плагины для WordPress

Для реализации всего функционала (услуги, каталог, мероприятия, карта) рекомендуется установить следующие плагины.

### 1. Для создания разделов и полей

*   **Custom Post Type UI (CPT UI):** Позволяет легко создавать новые типы контента ("Услуги", "Книги", "События").
*   **Advanced Custom Fields (ACF):** Позволяет добавлять к записям любые нужные поля ("Автор", "Год издания", "Цена" и т.д.).

### 2. Для мероприятий

*   **The Events Calendar:** Мощный плагин для создания и управления мероприятиями, с календарем и списками событий.

### 3. Для поиска по каталогу

*   **Relevanssi:** Улучшает стандартный поиск, позволяя искать по содержимому кастомных полей (например, по автору или ISBN).

### 4. Для интерактивной карты

*   **WP Go Maps:** Позволяет легко создавать кастомные карты Google Maps с метками и описаниями.

### 5. Для Headless CMS (Опционально)

*   **WPGraphQL:** Предоставляет GraphQL API, если вы решите использовать WordPress как бэкенд, а фронтенд разрабатывать на React/Next.js.

## ⚙️ Запуск статической React-версии

Если вы хотите запустить **исходную статическую версию сайта** (без WordPress):

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/elips0675-web/nlb-studio7.git
    cd nlb-studio7
    ```
2.  **Установите зависимости:**
    ```bash
    npm install
    ```
    *или*
    ```bash
    yarn install
    ```
3.  **Запустите в режиме разработки:**
    ```bash
    npm run dev
    ```
    *или*
    ```bash
    yarn dev
    ```
4.  Откройте `http://localhost:5173` (или другой порт, указанный Vite) в вашем браузере.

## ☁️ Развертывание статической React-версии

Проект настроен для развертывания на Firebase Hosting.

1.  **Соберите проект для продакшена:**
    ```bash
    npm run build
    ```
    *или*
    ```bash
    yarn build
    ```
    Эта команда создаст папку `dist` с оптимизированными статическими файлами.

2.  **Разверните на Firebase:**

    *Убедитесь, что у вас установлен Firebase CLI и вы вошли в свою учетную запись.*
    ```bash
    firebase deploy --only hosting
    ```
    Эта команда загрузит содержимое папки `dist` на хостинг Firebase.
