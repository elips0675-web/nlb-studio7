# План миграции с Vite/React на Next.js 15 (App Router)

Этот документ представляет собой исчерпывающий пошаговый план для полного переноса проекта с текущего стека (Vite, React, React Router) на современную архитектуру с Next.js 15 (App Router).

---

### **Фаза 0: Подготовка и очистка проекта**

*Цель: Подготовить проект к миграции, установив Next.js и удалив конфликтующие зависимости Vite.*

1.  **Установка Next.js:** Добавить `next` в зависимости проекта.
    *   **Команда:** `npm install next`
2.  **Обновление скриптов `package.json`:** Заменить скрипты `dev`, `build` на `next dev`, `next build` и `next start`.
3.  **Очистка `devDependencies`:** Удалить из `package.json` пакеты, специфичные для Vite, чтобы избежать конфликтов.
    *   **Пакеты для удаления:** `vite`, `@vitejs/plugin-react`, `vite-plugin-pwa`, `@tailwindcss/vite`.
    *   **Команда:** `npm uninstall vite @vitejs/plugin-react vite-plugin-pwa @tailwindcss/vite`
4.  **Удаление `react-router-dom`:** Next.js имеет встроенную систему роутинга, поэтому `react-router-dom` больше не нужен.
    *   **Команда:** `npm uninstall react-router-dom`
5.  **Создание конфигурации Next.js:** Создать базовый конфигурационный файл `next.config.mjs`.

---

### **Фаза 1: Создание новой структуры App Router**

*Цель: Создать базовую структуру папок и файлов, требуемую для работы Next.js App Router.*

1.  **Создание папки `app`:** Создать основную директорию `app/`, которая будет содержать все маршруты и страницы.
2.  **Создание корневого макета (`layout.tsx`):** Внутри `app/` создать файл `app/layout.tsx`. Это главный макет приложения, заменяющий `index.html`. В него необходимо перенести основную HTML-структуру (`<html>`, `<body>`), подключение шрифтов и глобальные провайдеры.
3.  **Создание главной страницы (`page.tsx`):** Внутри `app/` создать файл `app/page.tsx`. Это будет новая точка входа для главной страницы (`/`).
4.  **Настройка Tailwind CSS:** Обновить файл `tailwind.config.js`, добавив путь к новым файлам в `content`: '''./app/**/*.{js,ts,jsx,tsx,mdx}'''.
5.  **Перенос статичных ассетов:** Переместить все файлы из старой папки `public/` в новую (если они отличаются).

---

### **Фаза 2: Миграция маршрутов и страниц**

*Цель: Перенести существующие страницы и логику роутинга из React Router в файловую систему Next.js.*

1.  **Анализ текущих маршрутов:** Изучить файл `src/App.tsx` (или где определены маршруты), чтобы составить список всех существующих страниц.
2.  **Создание папок для маршрутов:** Для каждого старого маршрута (например, `/services`, `/about`) создать соответствующую папку в директории `app` (например, `app/services/`, `app/about/`).
3.  **Перенос контента страниц:** Скопировать содержимое React-компонентов старых страниц в новые файлы `page.tsx` внутри соответствующих папок.
4.  **Замена навигации:**
    *   Заменить все использования компонента `<Link>` из `react-router-dom` на `<Link>` из `next/link`.
    *   Заменить использование хука `useNavigate()` на `useRouter()` из `next/navigation` для программной навигации.

---

### **Фаза 3: Адаптация компонентов**

*Цель: Адаптировать существующие компоненты для работы в экосистеме Next.js, разделив их на серверные и клиентские.*

1.  **Перенос папки с компонентами:** Переместить папку с переиспользуемыми компонентами (`src/components`) в корень проекта.
2.  **Идентификация клиентских компонентов:** Проанализировать компоненты. Любой компонент, использующий хуки React (`useState`, `useEffect`, `useContext`) или обработчики событий (`onClick`, `onChange`), должен быть помечен директивой `"use client"` в самом начале файла.
3.  **Рефакторинг получения данных:** Пересмотреть компоненты, которые загружали данные через `useEffect`. По возможности перенести логику загрузки данных в серверные компоненты (страницы `page.tsx`), чтобы использовать преимущества Server-Side Rendering (SSR) и Server Components.

---

### **Фаза 4: Финальная очистка**

*Цель: Удалить все оставшиеся артефакты старой сборки Vite.*

1.  **Удаление старых файлов:**
    *   Удалить `index.html` из корня проекта.
    *   Удалить `vite.config.js`.
    *   Удалить всю папку `src`, так как ее содержимое было перенесено в `app` и `components`.
2.  **Проверка и тестирование:**
    *   Запустить сервер разработки (`npm run dev`) и тщательно проверить работоспособность всех страниц и компонентов.
    *   Проверить, что все ссылки работают, анимации проигрываются, и нет ошибок в консоли браузера и терминала.
