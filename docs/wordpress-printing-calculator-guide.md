# Руководство: Полиграфические услуги и калькулятор в WordPress

Этот документ описывает рекомендуемый подход для переноса раздела "Полиграфические услуги" и интерактивного калькулятора на WordPress-сайт.

---

## 1. Для раздела "Полиграфические услуги" (Printing Services)

Для создания и управления услугами печати в админ-панели WordPress лучше всего подходит комбинация из двух плагинов:

1.  **[Custom Post Type UI (CPT UI)](https://wordpress.org/plugins/custom-post-type-ui/)**
    *   **Что делает:** Позволяет создать новый, независимый тип контента. Вы можете создать тип записей "Услуги печати", который появится в левом меню административной панели.
    *   **Как использовать:** Установите плагин и создайте новый тип записей (например, с ярлыком `printing-service`).

2.  **[Advanced Custom Fields (ACF)](https://wordpress.org/plugins/advanced-custom-fields/)**
    *   **Что делает:** Позволяет "прикрепить" к созданным вами записям любые нужные поля (текст, числа, изображения, галереи и т.д.). Это позволяет структурировать контент, а не сваливать его в один редактор.
    *   **Как использовать:** Создайте новую группу полей и привяжите её к типу записей `printing-service`. Добавьте в нее поля, соответствующие вашей структуре данных в React:
        *   `subtitle` (Подзаголовок)
        *   `fullDescription` (Полное описание)
        *   `color` и `bgColor` (Цвета для оформления)
        *   `images` (Галерея или изображения)
        *   Поля для калькулятора: `base_price` (базовая цена), `price_per_page` (цена за страницу) и т.д.

**Результат:** Эта связка позволит вам удобно управлять услугами печати из админ-панели, а вашим разработчикам — легко выводить эти структурированные данные в шаблоне темы.

---

## 2. Для Калькулятора

Есть несколько способов перенести интерактивный калькулятор на WordPress.

### Вариант 1: Специализированный плагин (Рекомендуемый)

*   **Плагин:** **[Calculated Fields Form](https://wordpress.org/plugins/calculated-fields-form/)**
    *   **Что делает:** Это визуальный конструктор форм, который умеет производить математические расчеты на лету на основе данных, введенных пользователем.
    *   **Преимущества:** Не требует написания кода. Вы можете создать сложную логику расчета прямо в интерфейсе плагина и затем вставить готовый калькулятор на страницу с помощью шорткода.
    *   **Недостатки:** Внешний вид может потребовать дополнительной настройки с помощью CSS, чтобы соответствовать вашему дизайну.

### Вариант 2: Собственный JavaScript-код (Продвинутый)

*   **Что делать:** Взять существующую логику калькулятора из вашего React-приложения, адаптировать её (например, переписать на чистый JavaScript или подключить как изолированный React-компонент) и встроить напрямую в шаблон WordPress-темы.
*   **Преимущества:** Абсолютный контроль над внешним видом и поведением калькулятора. Он может динамически подгружать цены и параметры из тех самых ACF-полей, которые вы настроили для конкретной услуги.
*   **Недостатки:** Требует продвинутых навыков JavaScript-разработки и интеграции с WordPress.

### Заключение

- Для **услуг**: используйте связку **CPT UI + ACF**. Это золотой стандарт для создания кастомных типов контента в WordPress.
- Для **калькулятора**: начните с плагина **Calculated Fields Form**. Если его функционала или гибкости в настройке внешнего вида окажется недостаточно, переходите к интеграции собственного JavaScript-кода.
