# Руководство по миграции на Next.js 15 (App Router)

Этот документ описывает шаги, необходимые для перевода текущего приложения (Vite + React SPA) на современный стек с использованием Next.js 15 (App Router), TypeScript, Tailwind CSS, ShadCN UI и Framer Motion.

## 1. Подготовка и настройка проекта

1.  **Инициализация нового Next.js проекта:**

    ```bash
    npx create-next-app@latest my-app --typescript --tailwind --eslint
    ```

2.  **Установка дополнительных зависимостей:**

    ```bash
    npm install shadcn-ui framer-motion
    ```

3.  **Настройка ShadCN UI:**

    Инициализируйте ShadCN UI, чтобы настроить компоненты и файл `tailwind.config.js`:

    ```bash
    npx shadcn-ui@latest init
    ```

## 2. Миграция роутинга и страниц

Next.js 15 использует App Router, который основан на файловой системе. Вам нужно будет реорганизовать структуру страниц.

-   **`src/pages/` -> `src/app/`:** Переместите ваши страницы из `src/pages` в новую директорию `src/app`.
-   **`*.tsx` -> `page.tsx`:** Каждая страница должна быть представлена файлом `page.tsx` внутри своей собственной папки. Например, `src/pages/about.tsx` станет `src/app/about/page.tsx`.
-   **Layouts:** Создайте корневой layout `src/app/layout.tsx`, который будет общим для всего приложения.

## 3. Перенос компонентов

-   **Компоненты общего назначения:** Перенесите ваши переиспользуемые компоненты (например, из `src/app/components` в старом проекте) в `src/components` в новом проекте.
-   **Замена компонентов:** Замените базовые HTML-элементы и кастомные компоненты на аналоги из ShadCN UI, где это возможно. Это поможет унифицировать дизайн и улучшить доступность.

## 4. Обновление логики и хуков

-   **Получение данных:** Вместо `useEffect` и `fetch` в клиентских компонентах, используйте Server Components и асинхронные функции для получения данных на сервере. Это улучшит производительность и SEO.
-   **Хуки:** Некоторые хуки React (например, `useState`, `useEffect`) могут быть использованы только в клиентских компонентах. Пометьте такие компоненты директивой `"use client"`.

## 5. Стилизация и анимации

-   **Tailwind CSS:** Убедитесь, что ваш `tailwind.config.js` настроен правильно после инициализации ShadCN UI.
-   **Framer Motion:** Интегрируйте Framer Motion для анимаций. Вы можете обернуть ваши компоненты в `motion` элементы и использовать хуки, такие как `useAnimation`.
